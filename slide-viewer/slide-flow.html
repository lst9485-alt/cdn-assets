← 슬라이드 템플릿

원고 → 슬라이드 7단계

원고를 주면 최종 슬라이드가 나오기까지 이런 과정을 거칩니다

①
STEP 1
아웃라인 생성
manuscript.md → outline.py → tmp/outline.json
사람 확인 필요
▶

원고를 chunk(묶음) 단위로 나누고, 각 장면마다 어떤 분위기·신호(intent)를 줄지 자동으로 제안합니다. 사용자가 약 15분 동안 outline.json을 훑으면서 intent가 맞는지 확인하고 OK합니다.

template/outline.py tmp/outline.json
②
STEP 2
기획 (감독)
outline.json → 서브에이전트 5개 → tmp/plan_1~5.json
자동
▶

chunk마다 슬라이드 A/B/C 3가지 변형을 콘티처럼 짭니다. 각 슬라이드에 어떤 타입(T01~T47)을 쓸지, 텍스트 슬롯·step 수를 결정합니다. Claude 서브에이전트 5개가 동시에 chunk 하나씩 맡아서 처리합니다.

template/slide_plan.py tmp/plan_1.json ~ plan_5.json
③
STEP 3
작화
plan_*.json → 서브에이전트 5개 → tmp/chunk_1~5.txt
자동
▶

콘티(plan)를 보고 실제 텍스트·이미지·단계 정보를 채워 넣습니다. 결과는 정해진 형식의 텍스트 파일(chunk_*.txt)로 저장됩니다. 서브에이전트 5개가 병렬로 처리합니다.

tmp/chunk_1.txt ~ chunk_5.txt
④
STEP 4
병합 (Merge)
chunk_1~5.txt → merge_chunks.py → slide.md
자동
▶

5개 chunk 파일을 하나로 합치고 슬라이드 번호를 전체 기준으로 다시 매깁니다. 결과물인 slide.md가 이후 검증·HTML 생성의 원재료가 됩니다.

template/merge_chunks.py (L51 merge_and_renumber) template/compile_slidemd.py slide.md
⑤
STEP 5
검증
slide.md → checks/*.py → 오류 목록
자동
▶

슬라이드가 규칙에 맞는지 자동으로 검사합니다. 슬롯이 비었거나, A/B/C 변형의 step 수가 맞지 않거나, 필수 이미지가 없는 경우 등을 잡습니다. 문제가 있으면 다시 작화 단계로 돌아갑니다.

template/checks/ (검사기 약 20종) pipeline.py verify
⑥
STEP 6
HTML 생성
slide.md → generate-slides.py → {날짜}-slides.html
자동
▶

slide.md를 읽어서 각 슬라이드 타입에 맞는 HTML로 변환합니다. type-registry.json에 정의된 좌표·필드 정보를 사용합니다. 결과물은 260419-slides.html 같은 이름의 단일 HTML 파일입니다.

template/generate-slides.py references/type-registry.json references/snippets.md
⑦
STEP 7
CDN 배포
{날짜}-slides.html → sync-slide-viewer.sh → GitHub Pages
자동
▶

완성된 HTML을 cdn-assets 레포에 복사하고 GitHub에 push합니다. 1~2분 후 슬라이드 뷰어 URL에서 볼 수 있습니다. JS/CSS만 바뀌면 --assets-only 옵션으로 HTML 덮어쓰기 없이 배포합니다.

tools/scripts/sync-slide-viewer.sh https://lst9485-alt.github.io/cdn-assets/slide-viewer/

용어집
manuscript 사용자가 직접 쓴 원고 텍스트. 모든 것의 출발점.
outline 원고를 chunk 단위로 나누고 각 장면의 분위기(intent)를 제안한 JSON.
SlidePlan 슬라이드 콘티. 타입·슬롯·step 수를 A/B/C 3변형으로 미리 정해둔 것.
chunk 원고를 5등분한 묶음. 서브에이전트 하나가 chunk 하나씩 담당.
slide.md 5개 chunk를 합쳐서 만든 단일 마크다운. HTML 생성의 직접 재료.
verify slide.md를 자동으로 검사하는 단계. 오류 있으면 숫자와 종류를 리포트.
