슬라이드 UI 정리 시안

원티드 디자인시스템에서 가져올 만한 “정리 방식”만 눈으로 보는 테스트 페이지입니다.

이 페이지는 운영 에디터가 아닙니다. 링크 이동, 자동 점프, 저장 기능 없이 카드와 배지 정리 방향만 비교합니다.
템플릿 카드 정리 예시
T번호, 이름, 설명, 조건 배지를 같은 순서로 보여줍니다.
T01텍스트
말풍선+텍스트
질문을 던지고 바로 답을 보여줄 때 씁니다.
입력 21장
T07아이콘
아이콘행
아이콘 1~3개를 가로로 놓고 역할을 설명합니다.
입력 31~3개아이콘
T19경고
경고배너
위험, 실수, 주의할 포인트를 강하게 보여줍니다.
입력 2주의
T03카드/리스트
카드
여러 항목을 같은 무게로 나란히 비교합니다.
입력 41~5개
T09흐름
플로우
순서, 원인과 결과, 진행 과정을 설명합니다.
입력 21~5개
T37마무리
마지막정리
영상 끝에서 핵심만 다시 정리합니다.
입력 22~4개

지금 방식

  • 카드마다 정보 순서가 눈에 덜 들어옵니다.
  • 배지가 많아지면 어떤 조건이 중요한지 헷갈립니다.
  • 기능 설명과 개발용 정보가 섞여 보입니다.

정리 방식

  • T번호 → 이름 → 언제 쓰는지 → 조건 순서로 고정합니다.
  • 배지 색은 의미별로만 씁니다.
  • 기능은 추가하지 않고 보기만 정리합니다.
